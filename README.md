# SQL-DuckDB
Análise de Dados com SQL e DuckDB
Este projeto demonstra o uso de SQL analítico utilizando o DuckDB como motor de banco de dados embutido, com foco em consultas, agregações e organização de dados estruturados.
O objetivo principal é explorar boas práticas de escrita SQL em um ambiente leve, rápido e reproduzível.
Visão Geral:
O DuckDB é utilizado diretamente em ambiente Python/Jupyter, permitindo executar consultas SQL sobre arquivos locais sem necessidade de servidor.
O projeto simula um fluxo comum de análise de dados: ingestão, transformação e geração de métricas.
Tecnologias Utilizadas:
Python
DuckDB
SQL (CTEs, agregações, filtros e ordenações)
Jupyter Notebook
Estrutura do Projeto:
Carregamento de dados a partir de arquivo CSV
Criação de tabelas no DuckDB
Execução de consultas SQL para análise
Organização lógica das queries por objetivo analítico
Principais Operações SQL
SELECT, WHERE, ORDER BY
Funções de agregação (COUNT, AVG, SUM)
Agrupamentos com GROUP BY
Criação de métricas derivadas
Uso de CTEs para melhor legibilidade
Objetivo do Projeto:
Demonstrar domínio prático de SQL aplicado à análise de dados, com foco em clareza, performance e organização de consultas.
O projeto não tem foco em visualização, mas sim em lógica analítica e manipulação de dados.
Observações
O EDA completo e visualizações são tratados em projetos separados.
Este repositório foca exclusivamente na utilização de SQL com DuckDB.
